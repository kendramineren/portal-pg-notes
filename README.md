# portal-pg-notes

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库中的页面以静态文件形式存放，不依赖任何后端服务或特定域名绑定。每个页面均为自包含的 HTML 文件，可直接在浏览器中打开查看，也可通过 GitHub Pages 或其他静态托管服务进行部署。

## 项目简介

portal-pg-notes 的主要目的是提供一组独立、轻量的 HTML 页面归档。这些页面可能包含笔记、信息摘要、参考文档或其他静态内容，彼此之间没有强关联，各自独立使用。仓库结构简单，方便快速检索和引用。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 可选：存放页面引用的静态资源（如 CSS、图片、字体等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- `pages/`：核心目录，每个 `.html` 文件对应一个独立页面。文件名建议使用有意义的英文或拼音缩写，便于识别。
- `assets/`：如果页面需要共用样式或资源，可统一放置在此目录下。资源引用时使用相对路径，确保本地和线上均可正常加载。

## 页面归档说明

- 所有页面均为纯 HTML，不依赖外部 API 或数据库。
- 页面内容可能包含文本、表格、链接、图片等常见元素，但不包含用户登录、交互表单或动态数据加载。
- 归档的页面可能不定期更新或删除，但每个页面的基础结构保持稳定。如需引用，建议保存副本或关注仓库提交记录。
